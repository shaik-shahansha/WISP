Quickstart

Get a fully functional AI-driven device running in 5 minutes.

Prerequisites

You will need:

  • Python 3.9 or higher (python --version)
  • A free Telegram Bot token from @BotFather
  • A free Groq API key (or OpenRouter)
  • Any device you want to control (or your laptop — mock mode works everywhere)
No hardware? WISP includes a full mock/desktop simulator. Run wisp simulate to test without any physical device.

Step 1 — Install WISP

1
Install from PyPI
$ pip install wisp-ai

For Raspberry Pi, add the optional hardware extras:

$ pip install "wisp-ai[rpi]"
2
Verify installation
$ wisp version
WISP 0.1.0

Step 2 — Create a new project

The wisp new command scaffolds a project with the right file layout:

$ wisp new my_device
Created project: my_device/
  my_device/device.py
  my_device/config.json

You can also scaffold a sensors or ROS2 project:

$ wisp new robot --template ros2
Templates: basic (default), sensors (BME280 + BH1750), ros2 (ROS2Plugin auto-attached).

Step 3 — Edit config.json

Open my_device/config.json and fill in your API keys:

{
  "device_name": "my_device",
  "ai": {
    "provider": "groq",
    "api_key": "gsk_your_key_here",
    "model": "llama-3.3-70b-versatile"
  },
  "telegram": {
    "token": "123456:ABCdef...",
    "allowed_users": ["your_telegram_username"]
  }
}
Security: Never commit config.json to source control. Add it to .gitignore. You can use environment variables instead — WISP reads WISP_AI_API_KEY, WISP_TELEGRAM_TOKEN, etc.

Step 4 — Write your device class

Open my_device/device.py. The scaffold will already contain this pattern — customise it for your hardware:

from wisp import WispDevice, capability

class MyDevice(WispDevice):
    device_name = "my_device"
    device_description = "A smart device I control with Telegram."

    @capability
    def get_status(self) -> dict:
        """Return the device's current status."""
        return {"status": "online", "uptime": self.hardware.uptime()}

    @capability
    def set_led(self, state: str) -> dict:
        """Turn the LED on or off. state: 'on' | 'off'."""
        self.hardware.set_output("led", state == "on")
        return {"led": state}


if __name__ == "__main__":
    device = MyDevice.from_config("config.json")
    device.run()
Key rule: Every method decorated with @capability must have a docstring. The docstring is what the AI reads to understand what the capability does.

Step 5 — Run it

For quick testing without Telegram, use the interactive simulator:

$ cd my_device
$ wisp simulate
WISP 0.1.0  [simulator mode]
Device: my_device
Capabilities (2): get_status, set_led
Type a message or 'quit' to exit.
>>> what is the status?
✅  status: online, uptime: 42
>>> turn on the led
✅  led: on

When ready, deploy for real:

$ wisp run

This starts the Telegram long-poll loop. Your bot is now live and talking to your hardware.
