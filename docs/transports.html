Transports

WISP ships with three built-in transports: Telegram, CLI, and HTTP. Switch between them with one line in config.

Choosing a transport

TransportBest forRequires
telegramProduction — talk to devices from any phoneBot token, internet
cliDevelopment — interactive terminal REPLNothing
httpCustom integrations — REST API on local networkLocal network

Set the transport in config.json:

{
  "transport": "telegram"
}

Or override with an environment variable:

$ WISP_TRANSPORT=cli wisp run

Telegram transport

The recommended production transport. Uses long-polling — no webhook server, no public IP required. Zero external dependencies.

Setup

1
Create a bot with @BotFather

Open Telegram, find @BotFather, send /newbot, and copy the token.

2
Add token to config.json
{
  "telegram": {
    "token": "123456:ABCdef...",
    "allowed_users": ["your_username"],
    "poll_timeout": 30
  }
}
3
Run and message your bot
$ wisp run

Now open Telegram, find your bot, and send it any natural language message.

User whitelist

allowed_users is a list of Telegram usernames (without @). Only these users can send commands. Leave the list empty to allow everyone:

"allowed_users": []  // allow any Telegram user
Security: For devices on your local network or with physical actuators, always set an allowlist.

Config reference

KeyTypeDefaultDescription
tokenstring—Bot token from @BotFather. Required.
allowed_userslist[]Telegram usernames permitted to send commands.
poll_timeoutint30Long-poll timeout in seconds.
retry_delayint5Seconds to wait before retrying after an error.

CLI transport

An interactive REPL that runs in your terminal. Ideal for rapid development — no bot token needed.

$ wisp simulate
WISP 0.1.0  [simulator mode]
Device: my_device  |  Capabilities: read_temp, set_fan
>>> what is the temperature?
✅  temperature: 24.30, unit: C
>>> quit

Alternatively, set "transport": "cli" in config and run wisp run.

HTTP transport

A minimal HTTP server built on Python's stdlib http.server. Useful for integrating WISP into a larger system or calling it from a Home Assistant automation.

Endpoints

MethodPathDescription
POST/messageSend a natural-language command. Body: {"text": "...", "user": "..."}
GET/capabilitiesList all registered capabilities as JSON
GET/healthHealth check — returns {"ok": true}

Config

{
  "transport": "http",
  "_raw": {
    "transport_options": {
      "host": "0.0.0.0",
      "port": 8080
    }
  }
}

Example call

$ curl -X POST http://localhost:8080/message \
     -H "Content-Type: application/json" \
     -d '{"text": "read temperature", "user": "admin"}'
{"reply": "✅ temperature: 24.30, unit: C"}
