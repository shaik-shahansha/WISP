Hardware

WISP abstracts hardware through a HAL (Hardware Abstraction Layer) and auto-discovers I2C sensors at boot.

Hardware Abstraction Layer

The HAL (wisp.hardware.hal.HardwareLayer) auto-detects the platform at startup:

Detection orderConditionBackend
1sys.implementation.name == "micropython"MicroPython machine.I2C
2/proc/cpuinfo contains Raspberry Pismbus2 (install with wisp-ai[rpi])
3Anything else (laptop, CI, …)Mock — returns empty bytes, no errors
Mock mode lets you develop and test capabilities on any laptop without real hardware. All sensor reads return zero values and all GPIO writes are silently ignored.

HAL API

# Access hardware through self.hardware inside a WispDevice
hal = self.hardware

# I2C read/write
data = hal.i2c_read_register(addr=0x76, reg=0xF3, length=2)
hal.i2c_write_register(addr=0x76, reg=0xF4, data=bytes([0xB7]))

# GPIO (Raspberry Pi only; noop on other platforms)
hal.set_output("fan", True)
state = hal.get_output("fan")   # True / False
all_states = hal.output_states()  # {"fan": True, "led": False}

# Sensor instance (auto-discovered)
bme = hal.sensor("bme280")
print(bme.temperature, bme.humidity, bme.pressure)

I2C configuration

Configure bus and pins in config.json under "hardware":

{
  "hardware": {
    "i2c_bus": 1,
    "sda_pin": 21,
    "scl_pin": 22,
    "gpio_outputs": {
      "fan": 5,
      "led": 18
    }
  }
}
Platform defaults: On Raspberry Pi the bus defaults to 1 (pins 3/5). On ESP32 you must specify sda_pin and scl_pin.

I2C auto-discovery

At boot, I2CScanner probes all I2C addresses and matches them against a built-in lookup table. Matched sensors are instantiated and registered as capabilities automatically — no extra code needed.

Supported sensors

SensorDefault addressCapability nameData returned
BME2800x76 / 0x77 read_bme280 temperature, humidity, pressure
BH17500x23 / 0x5C read_bh1750 lux
SHT310x44 / 0x45 read_sht31 temperature, humidity
MPU60500x68 / 0x69 read_mpu6050 ax, ay, az, gx, gy, gz
SSD13060x3C / 0x3D display_text OLED — writes text to display

Wiring guide

ESP32 ─── BME280
  3V3 ──── VCC
  GND ──── GND
  GPIO21 ─ SDA
  GPIO22 ─ SCL

Raspberry Pi ─── BME280
  3V3     ──── VCC
  GND     ──── GND
  GPIO2 (SDA) ─ SDA
  GPIO3 (SCL) ─ SCL

All other supported sensors (BH1750, SHT31, MPU6050, SSD1306) wire identically — VCC / GND / SDA / SCL to the same pins.

Custom sensor drivers

If your sensor is not in the built-in list, write a driver and add it to the scanner lookup table:

from wisp.hardware.hal import HardwareLayer

class MyDAC:
    def __init__(self, hal: HardwareLayer, addr: int):
        self.hal = hal
        self.addr = addr

    def set_voltage(self, mv: int) -> None:
        hi = (mv >> 8) & 0x0F
        lo = mv & 0xFF
        self.hal.i2c_write_register(self.addr, 0x40, bytes([hi, lo]))

Then register it manually in your device's __init__ or _auto_discover override.

Platform notes

🔲

ESP32

Flash MicroPython 1.21+ and copy the device/ folder with mpremote. Use config.json for WiFi credentials.

🍓

Raspberry Pi

Enable I2C with raspi-config, then install pip install "wisp-ai[rpi]". GPIO via RPi.GPIO.

💻

Desktop / CI

HAL falls back to mock mode. All sensor reads return zeros. GPIO calls are silently ignored. Tests run everywhere.

🤖

ROS2 Robot

Hardware is ROS2 topics, not I2C. Use ROS2Plugin instead of direct HAL access — it maps topics to capabilities.
