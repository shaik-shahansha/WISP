API Reference

Complete reference for every public class and decorator in wisp.

WispDevice

The base class for every WISP device. Subclass it, decorate methods with @capability, call .run().

class wisp.WispDevice

Base class. Powered by WispDeviceMeta which auto-collects all @capability-decorated methods at class creation time.

Class attributeTypeDescription
device_namestrRequired. Machine-readable identifier sent to the AI.
device_descriptionstrOptional description used to help the AI understand context.
classmethod WispDevice.from_config(path: str) → WispDevice

Load a WispConfig from a JSON file and return an initialised device instance.

device = MyDevice.from_config("config.json")
method device.run() → None

Start the configured transport (Telegram, CLI, or HTTP). This call blocks the current thread.

The transport is read from config.transport. If the environment variable WISP_TRANSPORT=cli is set, the CLI transport overrides the config — useful for wisp simulate.

method device.use(plugin: WispPlugin) → WispDevice

Attach a plugin to the device. Returns self for chaining.

device.use(ROS2Plugin()).use(MyPlugin())
method device.process_message(text: str, user: str) → str

The core pipeline: build grounded prompt → call AI → parse JSON → execute capability → format reply.

Used internally by all transports. You can call it directly in tests or custom integrations.

method device._auto_discover() → None

Called at boot. Scans I2C bus and registers sensor capabilities automatically. Override if you want custom boot logic.

@capability

Decorator that marks a method as an AI-callable capability.

decorator @wisp.capability

Works bare or with arguments:

# Bare — name inferred from function name
@capability
def read_temperature(self) -> dict:
    """Read temperature from BME280."""
    ...

# With args — override name or add tags
@capability(name="temp", tags=["sensors"])
def read_temperature(self) -> dict:
    ...
Docstring required. The docstring is injected into the AI system prompt. Without it, the AI won't understand what the capability does.

Return type

Capability methods must return a dict. The dict is serialised to a human-readable string and sent back to the user.

Parameters

WISP inspects the function signature to extract parameter names and type hints. These are included in the AI schema so the AI knows what arguments to supply.

@capability
def set_fan(self, state: str, speed: int = 50) -> dict:
    """Set the fan. state: 'on'|'off'. speed: 0-100."""
    ...

WispPlugin

abstract class wisp.WispPlugin

Base class for all plugins. Subclass and implement attach().

from wisp import WispPlugin, WispDevice, CapabilitySpec

class MyPlugin(WispPlugin):
    def attach(self, device: WispDevice) -> None:
        # Register capabilities dynamically
        spec = CapabilitySpec(
            name="ping",
            description="Ping the network.",
            method=self._do_ping,
        )
        device.registry.register(spec)

    def _do_ping(self) -> dict:
        return {"ping": "pong"}
